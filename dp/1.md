# Problema 1 Programación Dinámica

## Link
https://leetcode.com/problems/longest-increasing-subsequence/description/

## Enunciado
Dado un arreglo de enteros `nums`, devuelve la longitud de la subsecuencia estrictamente creciente más larga.

### Ejemplo 1:
**Input:** `nums = [10,9,2,5,3,7,101,18]`
**Output:** `4`
**Explicación:** La subsecuencia creciente más larga es `[2,3,7,101]`, por lo tanto, la longitud es 4.

### Ejemplo 2:
**Input:** `nums = [0,1,0,3,2,3]`
**Output:** `4`

### Ejemplo 3:
**Input:** `nums = [7,7,7,7,7,7,7]`
**Output:** `1`

### Restricciones:
*   `1 <= nums.length <= 2500`
*   `-10^4 <= nums[i] <= 10^4`

## Explicación
El algoritmo resuelve este problema usando programación dinámica. La idea principal se basa en el principio de **subestructura óptima**: la solución para encontrar la subsecuencia más larga se puede construir a partir de las soluciones de subproblemas más pequeños.

Para lograrlo, se siguen estos pasos:
1.  **Inicialización**: Se crea un arreglo auxiliar `dp` del mismo tamaño que `nums`. Cada posición `dp[i]` almacenará la longitud de la subsecuencia creciente más larga que termina *exactamente* en el elemento `nums[i]`. Inicialmente, todos los valores de `dp` se establecen en `1`, ya que cada número por sí solo constituye una subsecuencia de longitud 1.
2.  **Construcción de la solución**: Se utilizan dos bucles anidados para recorrer el arreglo.
    *   El bucle externo (`i`) avanza por cada elemento del arreglo `nums`.
    *   El bucle interno (`j`) revisa todos los elementos que vienen *antes* de `nums[i]`.
3.  **Actualización de `dp`**: Dentro del bucle interno, se compara `nums[i]` con `nums[j]`. Si `nums[i]` es mayor que `nums[j]`, significa que podemos extender la subsecuencia que terminaba en `nums[j]`. Por lo tanto, se actualiza `dp[i]` al máximo entre su valor actual y `dp[j] + 1`. Este paso asegura que siempre conservemos la longitud de la subsecuencia más larga encontrada hasta ese punto para `nums[i]`.
4.  **Resultado final**: Después de que los bucles han terminado, el arreglo `dp` contiene las longitudes de las subsecuencias crecientes más largas para cada punto final posible. La respuesta al problema es simplemente el valor máximo en todo el arreglo `dp`, ya que la subsecuencia más larga puede terminar en cualquier posición.

## Código
```cpp
class Solution { 
public:
    int lengthOfLIS(vector<int>& nums) {
        if (nums.empty()) return 0;
        
        int n = nums.size();
        vector<int> dp(n, 1);
        
        for (int i = 0; i < n; ++i) {
            for (int j = 0; j < i; ++j) {
                if (nums[i] > nums[j] && dp[i] < dp[j] + 1) {
                    dp[i] = dp[j] + 1;
                }
            }
        }        
        return *max_element(dp.begin(), dp.end());
    }
};
```
