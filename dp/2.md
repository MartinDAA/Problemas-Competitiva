# Problema 2 Programación Dinámica

## Link
https://leetcode.com/problems/coin-change/description/

## Enunciado
You are given an integer array coins representing coins of different denominations and an integer amount representing a total amount of money.

Return the fewest number of coins that you need to make up that amount. If that amount of money cannot be made up by any combination of the coins, return -1.

You may assume that you have an infinite number of each kind of coin.


Example 1:

Input: coins = [1,2,5], amount = 11
Output: 3
Explanation: 11 = 5 + 5 + 1
Example 2:

Input: coins = [2], amount = 3
Output: -1
Example 3:

Input: coins = [1], amount = 0
Output: 0
 

Constraints:

1 <= coins.length <= 12
1 <= coins[i] <= 231 - 1
0 <= amount <= 104

## Explicación

```cpp
class Solution {
public:
    int coinChange(vector<int>& coins, int amount) {
        
        vector<int> memo(amount + 1, -2);
        return solve(coins, amount, memo);
    }
private:
    int solve(vector<int>& coins, int rem, vector<int>& memo) {
        if (rem < 0) return -1; 
        if (rem == 0) return 0;  
        if (memo[rem] != -2) return memo[rem]; 
        
        int min_count = INT_MAX;
        
        for (int coin : coins) {
            int res = solve(coins, rem - coin, memo);
            if (res >= 0 && res < min_count) {
                min_count = 1 + res;
            }
        }
        
        memo[rem] = (min_count == INT_MAX) ? -1 : min_count;
        return memo[rem];
    }
};
```
El algoritmo resuelve el problema del cambio de monedas utilizando programación dinámica con un enfoque Top-Down, o recursión con memoización, para encontrar la mínima cantidad posible de monedas que sumen el monto objetivo (`amount`). Para evitar el cálculo redundante de subproblemas y prevenir un tiempo de ejecución excesivo, el código utiliza un arreglo auxiliar `memo` inicializado en `-2`, un valor estratégico que permite diferenciar entre un estado aún no calculado y uno que ya se evaluó como imposible (representado por el valor `-1`). La función recursiva `solve` descompone el problema restando sistemáticamente cada moneda disponible al monto restante (`rem`), utilizando como casos base que una cantidad de `0` requiere exactamente cero monedas, mientras que una cantidad negativa indica una ruta o combinación inválida. En cada paso, antes de realizar cualquier iteración, el algoritmo verifica si la respuesta para esa cantidad ya existe en la memoria; de no ser así, explora todas las ramas posibles con un ciclo, selecciona el subproblema válido que devuelva el menor número de monedas, le suma una unidad (la moneda que se está probando en ese momento) y guarda este resultado óptimo en `memo` antes de retornarlo, garantizando así una solución completa y altamente eficiente.
