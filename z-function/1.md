## Enunciado
### D. Common Divisors
time limit per test
2 seconds
memory limit per test
256 megabytes

Vasya has recently learned at school what a number's divisor is and decided to determine a string's divisor. Here is what he came up with.

String a is the divisor of string b if and only if there exists a positive integer x such that if we write out string a consecutively x times, we get string b. For example, string "abab" has two divisors — "ab" and "abab".

Now Vasya wants to write a program that calculates the number of common divisors of two strings. Please help him.
Input

The first input line contains a non-empty string s1.

The second input line contains a non-empty string s2.

Lengths of strings s1 and s2 are positive and do not exceed 105. The strings only consist of lowercase Latin letters.
Output

Print the number of common divisors of strings s1 and s2. 
## Explicación

```cpp
vector<int> calcular_arreglo_z(string cadena) {
    int n = cadena.length();
    vector<int> z(n, 0);
    int izq = 0, der = 0;

    for (int i = 1; i < n; ++i) {
        if (i <= der) {
            z[i] = min(der - i + 1, z[i - izq]);
        }
        while (i + z[i] < n && cadena[z[i]] == cadena[i + z[i]]) {
            z[i]++;
        }
        if (i + z[i] - 1 > der) {
            izq = i;
            der = i + z[i] - 1;
        }
    }
    return z;
}  return 0;
}
```
Esta función hace uso del algoritmo z que calcula un arreglo donde cada posición z\[i] guarda cuántos caracteres seguidos, empezando desde el índice i, son exactamente iguales al inicio de la cadena. Para avanzar rápido sin repetir comparaciones, el código usa una o caja con los límites izq y der que recuerda el tramo coincidente más largo que ya se ha revisado, si el índice actual i cae dentro de esa zona, se copia la información ya calculada usando la función min. Luego, un bucle while compara los caracteres uno a uno hacia adelante para ver si la coincidencia puede ser todavía más larga y, si se logra superar el límite der, se actualizan los bordes de la ventana antes de devolver el arreglo final.

Aquí se calcula el Máximo Común Divisor gcd entre los tamaños de ambas cadenas para definir la longitud de la subcadena más grande que podría dividirlas a ambas. Con ese tamaño, se extrae ese pedazo inicial de s1 para usarlo como el único candidato a divisor. Finalmente, se procesa este bloque con el Algoritmo Z para analizar sus repeticiones internas y se inicializa en 0 el contador que acumulará los divisores válidos hallados.

Este bucle evalúa cada longitud posible \(len) para ver si el prefijo de ese tamaño es un divisor válido del bloque candidato. Primero, verifica que la longitud divida exactamente al tamaño máximo \(tam_mcd % len == 0). Luego, si el prefijo abarca toda la cadena se cuenta como válido, y si es más pequeño, se usa el arreglo Z para revisar si ese trozo se repite de forma idéntica hasta el final \(z[len] == tam_mcd - len). Cada vez que se cumple esta condición se suma uno al contador, el cual se imprime al terminar el recorrido.
