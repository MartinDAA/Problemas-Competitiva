## Enunciado
### E. Compress Words
time limit per test
1 second
memory limit per test
256 megabytes

Amugae has a sentence consisting of n words. He want to compress this sentence into one word. Amugae doesn't like repetitions, so when he merges two words into one word, he removes the longest prefix of the second word that coincides with a suffix of the first word. For example, he merges "sample" and "please" into "samplease".

Amugae will merge his sentence left to right (i.e. first merge the first two words, then merge the result with the third word and so on). Write a program that prints the compressed word after the merging process ends.
Input

The first line contains an integer n (1≤n≤105), the number of the words in Amugae's sentence.

The second line contains n words separated by single space. Each words is non-empty and consists of uppercase and lowercase English letters and digits ('A', 'B', ..., 'Z', 'a', 'b', ..., 'z', '0', '1', ..., '9'). The total length of the words does not exceed 106.
Output

In the only line output the compressed word after the merging process ends as described in the problem.
## Explicación

```cpp
vector<int> calcular_arreglo_z(string cadena) {
    int n = cadena.length();
    vector<int> z(n, 0);
    int izq = 0, der = 0;

    for (int i = 1; i < n; ++i) {
        if (i <= der) {
            z[i] = min(der - i + 1, z[i - izq]);
        }
        while (i + z[i] < n && cadena[z[i]] == cadena[i + z[i]]) {
            z[i]++;
        }
        if (i + z[i] - 1 > der) {
            izq = i;
            der = i + z[i] - 1;
        }
    }
    return z;
}
```
Uso del algoritmo z, explicado anteriormente, que calcula un arreglo donde cada posición z\[i] guarda cuántos caracteres seguidos, empezando desde el índice i, son exactamente iguales al inicio de la cadena

Primero, se obtienen los tamaños de ambas cadenas, luego mediante la función min, se define que solo tomaremos del final de resultado una cantidad de letras igual o menor al tamaño de la nueva_palabra, ya que el union no puede ser más grande que la palabra entrante. Con esa cantidad, se extrae el trozo_final utilizando la función substr. Luego, se crea la cadena combinada uniendo la nueva_palabra, un separador # (para evitar falsas coincidencias), y el fragmento del final que acabamos de cortar. Finalmente, se envía esta combinación al Algoritmo Z para descubrir exactamente dónde y cuánto se repiten los caracteres entre ambas secciones.

En este bloque se determina cuántas letras se repiten para unir las palabras de forma correcta. Se define la variable inicio_trozo para saltarse la palabra nueva junto con el carácter # y empezar a revisar directamente en la sección del fragmento final. El bucle for recorre ese tramo buscando la primera posición donde la coincidencia del prefijo llegue exactamente hasta el final de la cadena combinada \(j + z\[j] == tam_comb); al encontrarla, se guarda esa longitud en max_solapamiento y se rompe el ciclo inmediatamente por ser el choque más grande posible. Finalmente, se toma la palabra nueva, se le quita ese pedazo repetido usando substr, y lo restante se concatena directamente a la variable resultado para luego imprimir la palabra completamente comprimida al salir del bucle principal.
