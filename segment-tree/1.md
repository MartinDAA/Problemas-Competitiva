# Problema 1 Segment Tree

## Link
[Codeforces - 356A. Knight Tournament](https://codeforces.com/problemset/problem/356/A)
## Enunciado
### A. Knight Tournament
time limit per test
3 seconds
memory limit per test
256 megabytes

Hooray! Berl II, the king of Berland is making a knight tournament. The king has already sent the message to all knights in the kingdom and they in turn agreed to participate in this grand event.

As for you, you're just a simple peasant. There's no surprise that you slept in this morning and were late for the tournament (it was a weekend, after all). Now you are really curious about the results of the tournament. This time the tournament in Berland went as follows:

    There are n knights participating in the tournament. Each knight was assigned his unique number — an integer from 1 to n.
    The tournament consisted of m fights, in the i-th fight the knights that were still in the game with numbers at least li and at most ri have fought for the right to continue taking part in the tournament.
    After the i-th fight among all participants of the fight only one knight won — the knight number xi, he continued participating in the tournament. Other knights left the tournament.
    The winner of the last (the m-th) fight (the knight number xm) became the winner of the tournament. 

You fished out all the information about the fights from your friends. Now for each knight you want to know the name of the knight he was conquered by. We think that the knight number b was conquered by the knight number a, if there was a fight with both of these knights present and the winner was the knight number a.

Write the code that calculates for each knight, the name of the knight that beat him.
Input

The first line contains two integers n, m (2 ≤ n ≤ 3·105; 1 ≤ m ≤ 3·105) — the number of knights and the number of fights. Each of the following m lines contains three integers li, ri, xi (1 ≤ li < ri ≤ n; li ≤ xi ≤ ri) — the description of the i-th fight.

It is guaranteed that the input is correct and matches the problem statement. It is guaranteed that at least two knights took part in each battle.
Output

Print n integers. If the i-th knight lost, then the i-th number should equal the number of the knight that beat the knight number i. If the i-th knight is the winner, then the i-th number must equal 0.

## Explicación

```cpp
const int MAXN = 300005;

int arbol[4 * MAXN];
int respuesta[MAXN];
```
Esta parte sirve para apartar el espacio antes de arrancar el programa. Guarda una constante con el límite máximo de caballeros que pide el problema, crea el arreglo arbol con cuatro veces ese tamaño para asegurar todos los niveles \(padres e hijos) de la estructura del Segment Tree.

```cpp
void actualizar(int nodo, int inicio, int fin, int l, int r, int ganador) {
    if (inicio > r || fin < l) {
        return;
    }

    if (l <= inicio && fin <= r) {
        arbol[nodo] = ganador; 
        return;
    }

    if (arbol[nodo] != 0) {
        arbol[2 * nodo] = arbol[nodo];
        arbol[2 * nodo + 1] = arbol[nodo];
        arbol[nodo] = 0;
    }

    int medio = inicio + (fin - inicio) / 2;
    actualizar(2 * nodo, inicio, medio, l, r, ganador);
    actualizar(2 * nodo + 1, medio + 1, fin, l, r, ganador);
}
```
Esta función registra al ganador de una pelea en un rango específico dentro del Segment Tree. Primero, comprueba si el territorio del nodo actual \(definido por inicio y fin) no choca con el rango de la batalla \(l y r); si no hay cruce \(inicio > r || fin < l), se detiene con un return. Luego, si la pelea cubre por completo el rango del nodo \(l <= inicio && fin <= r), guarda directamente el valor de ganador en la posición arbol[nodo] y detiene la búsqueda. En caso de que la cobertura sea parcial, revisa si arbol[nodo] tiene un ganador guardado previamente; si es así, transfiere esa información a sus dos nodos hijos \(2 * nodo y 2 * nodo + 1) y limpia el valor del padre poniendo arbol\[nodo] = 0. Finalmente, calcula el punto medio del rango para dividirlo en dos mitades y se ejecuta de forma recursiva para actualizar de manera precisa tanto el lado izquierdo \(2 * nodo) como el derecho \(2 * nodo + 1).

```cpp
void empujar_todo(int nodo, int inicio, int fin) {
    if (inicio == fin) {

        respuesta[inicio] = arbol[nodo];
        return;
    }

    if (arbol[nodo] != 0) {
        arbol[2 * nodo] = arbol[nodo];
        arbol[2 * nodo + 1] = arbol[nodo];
    }

    int medio = inicio + (fin - inicio) / 2;
    empujar_todo(2 * nodo, inicio, medio);
    empujar_todo(2 * nodo + 1, medio + 1, fin);
}
```
Esta función se ejecuta al final del programa para introducir toda la información acumulada en el arbol y guardarla de forma definitiva. Primero, comprueba si el rango actual llegó a su punto mínimo \(inicio == fin); de ser así, significa que alcanzó la hoja de un caballero individual y guarda el valor de arbol\[nodo] en el arreglo plano respuesta\[inicio] para luego salir con un return. Si aún no es una hoja, revisa si el nodo actual \(arbol\[nodo] != 0) contiene un ganador acumulado; si se cumple, transfiere por seguridad esa información a sus dos posiciones hijas \(arbol\[2 * nodo] y arbol\[2 * nodo + 1]) para que los datos sigan bajando. Finalmente, calcula el punto medio del rango actual y se llama a sí misma de forma recursiva para repetir este proceso de limpieza tanto en la mitad izquierda \(2 * nodo) como en la mitad derecha \(2 * nodo + 1) hasta procesar a todos los caballeros.

```cpp
    int n, m;
    cin >> n >> m;

    vector<int> L(m), R(m), X(m);
    for (int i = 0; i < m; ++i) {
        cin >> L[i] >> R[i] >> X[i];
    }
```
Se introducen los datos, primero lee n cantidad de caballeros y m número de peleas. Luego crea tres vectores \(L, R y X) de tamaño m , mediante un bucle for que corre m veces, guarda en cada posición i el inicio del combate en L\[i], el fin en R\[i], y el caballero ganador en X\[i].


```cpp
    for (int i = m - 1; i >= 0; --i) {

        if (L[i] < X[i]) {
            actualizar(1, 1, n, L[i], X[i] - 1, X[i]);
        }
        if (X[i] < R[i]) {
            actualizar(1, 1, n, X[i] + 1, R[i], X[i]);
        }
    }
```
Este bucle for procesa todas las peleas al revés, desde la última \(m - 1) hasta la primera 0. En cada iteración, el caballero ganador X\[i] elimina a todos los demás dentro del rango desde L\[i] hasta R\[i]. Como el ganador no se elimina a sí mismo, el programa divide la actualización del Segment Tree en dos partes: primero borra al grupo de la izquierda desde L\[i] hasta X\[i] - 1 \(si existe), y luego borra al grupo de la derecha desde X\[i] + 1 hasta R\[i] \(si existe), pasando siempre el valor de X\[i] como el ejecutor de la derrota.

```cpp
    empujar_todo(1, 1, n);

    for (int i = 1; i <= n; ++i) {
        cout << respuesta[i] << (i == n ? "" : " ");
    }
```
Se ejecuta la función empujar_todo(1, 1, n) iniciando desde la raíz nodo 1 para arrastrar en cascada todos los datos que quedaron guardados a mitad del árbol y depositarlos en las hojas correspondientes a cada caballero. Luego, un bucle for recorre el arreglo respuesta desde el caballero 1 hasta n imprimiendo el ID de quién eliminó a cada uno, usando un operador condicional para separar los números con un espacio simple excepto en el último elemento (i == n).
