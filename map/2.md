## Enunciado
### D. Epic Transformation
time limit per test
2 seconds
memory limit per test
256 megabytes

You are given an array a of length n consisting of integers. You can apply the following operation, consisting of several steps, on the array a zero or more times:

    you select two different numbers in the array ai and aj;
    you remove i-th and j-th elements from the array. 

For example, if n=6 and a=[1,6,1,1,4,4], then you can perform the following sequence of operations:

    select i=1,j=5. The array a becomes equal to [6,1,1,4];
    select i=1,j=2. The array a becomes equal to [1,4]. 

What can be the minimum size of the array after applying some sequence of operations to it?
Input

The first line contains a single integer t (1≤t≤104). Then t test cases follow.

The first line of each test case contains a single integer n (1≤n≤2⋅105) is length of the array a.

The second line of each test case contains n integers a1,a2,…,an (1≤ai≤109).

It is guaranteed that the sum of n over all test cases does not exceed 2⋅105.
Output

For each test case, output the minimum possible size of the array after applying some sequence of operations to it.
## Explicación

```cpp
void resolver() {
    int n;
    cin >> n;

    map<int, int> frecuencias;
    int f_max = 0;

    for (int i = 0; i < n; ++i) {
        int numero;
        cin >> numero;
        frecuencias[numero]++;

        f_max = max(f_max, frecuencias[numero]);
    }
```
Para contar los elementos se declara un map llamado frecuencias donde se guarda cada número del arreglo como clave y su cantidad de repeticiones como valor. Luego, a medida que se lee cada número, se accede directamente a su posición en el map mediante frecuencias\[numero]++ para incrementar en uno su contador individual. Finalmente, para evitar recorrer todo el mapa al terminar la lectura, se emplea la función max en tiempo real, lo que permite capturar y actualizar en la variable f_max la frecuencia más alta del arreglo de forma inmediata.


```cpp
    if (f_max > n - f_max) {
        cout << 2 * f_max - n << "\n";
    } 
    else {
        cout << n % 2 << "\n";
    }
```



Luego se revisa si el número que más se repite aparece tantas veces que es mayor que todo el resto de los números juntos (n - f_max). Si este número  le gana a todos los demás combinados, es imposible borrarlo por completo; por lo tanto, usamos todos los otros números distintos para eliminar la mayor cantidad posible de copias de este, lo que por una simple resta nos deja un sobrante exacto de 2 * f_max - n. Por otro lado, si ningún número es tan grande como para ganarle al resto, significa que se puede juntar los elementos en parejas diferentes de forma inteligente para que se eliminen entre sí; bajo esta lógica, el arreglo quedará totalmente vacío 0 si el total de elementos es par, o nos sobrará un único número 1 si el total es impar, lo cual se calcula rápidamente con la operación n % 2.

```cpp
int main() {
    int casos;
    if (cin >> casos) {
        while (casos--) {
            resolver();
        }
    }
    return 0;
}
```

Aqui se introduce la cantidad de casos y luego se establece un bucle while que repite la cantidad del numero de casos donde se ejecuta la funcion resolver.
