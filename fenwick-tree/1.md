# Problema 1 Fenwick Tree

## Link
https://leetcode.com/problems/count-of-smaller-numbers-after-self/description/

## Enunciado
Given an integer array nums, return an integer array counts where counts[i] is the number of smaller elements to the right of nums[i].

Example 1:

Input: nums = [5,2,6,1]
Output: [2,1,1,0]
Explanation:
To the right of 5 there are 2 smaller elements (2 and 1).
To the right of 2 there is only 1 smaller element (1).
To the right of 6 there is 1 smaller element (1).
To the right of 1 there is 0 smaller element.
Example 2:

Input: nums = [-1]
Output: [0]
Example 3:

Input: nums = [-1,-1]
Output: [0,0]
 

Constraints:

1 <= nums.length <= 105
-104 <= nums[i] <= 104

## Explicación

Esta solución utiliza un **Fenwick Tree (Binary Indexed Tree)** para contar de manera eficiente cuántos elementos menores existen a la derecha de cada número procesando el arreglo al revés (de derecha a izquierda). Como el árbol de Fenwick no acepta índices negativos ni valores nulos en sus operaciones estándar, el algoritmo primero calcula el valor mínimo (`shift`) y el valor máximo del arreglo para "normalizar" los datos; esto desplaza todo el rango de números para que el menor de todos empiece exactamente en un terreno seguro mayor o igual a cero. Al recorrer el arreglo en reversa, cada número `nums[i]` consulta el total acumulado en el árbol hasta la posición de su valor desplazado (`fT.find(nums[i] - shift)`), obteniendo instantáneamente cuántos números menores ya han sido registrados a su derecha, y acto seguido se inserta a sí mismo en el árbol (`fT.add`) para que los elementos que están más a la izquierda lo tengan en cuenta en sus propios cálculos.
