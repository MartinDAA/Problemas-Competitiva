# Problema 1 Trie: Palindrome Pairs

## Link
[Leetcode - 336. Palindrome Pairs](https://leetcode.com/problems/palindrome-pairs/description/)

## Enunciado
You are given a 0-indexed array of unique strings `words`.

A palindrome pair is a pair of integers `(i, j)` such that:

* `0 <= i, j < words.length`,
* `i != j`, and
* `words[i] + words[j]` (the concatenation of the two strings) is a palindrome.

Return an array of all the palindrome pairs of words.

You must write an algorithm with `O(sum of words[i].length)` runtime complexity.

 

Example 1:

Input: `words = ["abcd","dcba","lls","s","sssll"]`\
Output: `[[0,1],[1,0],[3,2],[2,4]]`\
Explanation: The palindromes are `["abcddcba","dcbaabcd","slls","llssssll"]`

Example 2:

Input: `ords = ["bat","tab","cat"]`\
Output: `[[0,1],[1,0]]`\
Explanation: The palindromes are `["battab","tabbat"]`

Example 3:

Input: `words = ["a",""]`\
Output: `[[0,1],[1,0]]`\
Explanation: The palindromes are `["a","a"]`
 

Constraints:

`1 <= words.length <= 5000`\
`0 <= words[i].length <= 300`\
`words[i]` consists of lowercase English letters.

## Explicación

### Enfoque

El problema pide encontrar todos los pares de índices `(i, j)` tales que la concatenación de `words[i]` y `words[j]` forme un palíndromo. La solución emplea un **trie** que almacena las palabras invertidas, lo que permite verificar en tiempo lineal si una palabra y otra pueden complementarse para formar un palíndromo.

### Algoritmo

1. **Construcción del trie (función `add`)**  
   - Se inserta cada palabra `s` en el trie recorriéndola desde el último carácter hasta el primero (es decir, se almacena la palabra invertida).  
   - En cada nodo del trie se guarda:
     - `index`: el índice de la palabra que termina exactamente en ese nodo (si existe).
     - `palindromeIndexes`: una lista de índices de palabras para las cuales el sufijo desde la posición actual hasta el final de la palabra original es palíndromo.  
   - Durante la inserción, para cada prefijo de la palabra invertida (que corresponde a un sufijo de la palabra original), se verifica si el resto de la palabra original (desde el inicio hasta la posición `j`) es palíndromo. Si lo es, se añade el índice de la palabra actual a `palindromeIndexes` del nodo.

2. **Búsqueda de pares (función `search`)**  
   - Para cada palabra `s` (con índice `i`), se recorre el trie siguiendo los caracteres de `s` en orden normal.  
   - En cada paso, si el nodo actual contiene un índice de palabra (`index != -1`) y el sufijo restante de `s` es palíndromo, entonces se forma un par `(i, index)`.  
   - Al final del recorrido de `s`, se añaden todos los pares `(i, j)` donde `j` está en `palindromeIndexes` del nodo final (esto cubre los casos en que la palabra `s` es más corta o igual que la palabra almacenada en el trie).  
   - Se omite el caso `i == j` para evitar pares con la misma palabra.

3. **Verificación de palíndromos**  
   - La función `isPalindrome` comprueba si una subcadena es palíndromo mediante dos punteros.

### Complejidad

- **Tiempo**: `O(n * L^2)` en el peor caso, donde `n` es el número de palabras y `L` es la longitud máxima de una palabra. La verificación de palíndromos durante la inserción y la búsqueda puede tomar `O(L)` por carácter, resultando en `O(L^2)` por palabra. En la práctica, con las restricciones del problema, es aceptable.
- **Espacio**: `O(n * L)` para almacenar el trie.

### Puntos clave

- El uso de un trie de sufijos (palabras invertidas) permite emparejar eficientemente una palabra con otra que pueda completar un palíndromo.
- El almacenamiento de `palindromeIndexes` en cada nodo evita tener que recorrer todo el trie para cada palabra, reduciendo el número de comparaciones.
- La solución maneja correctamente casos borde como palabras vacías y palabras que son palíndromos por sí mismas.
