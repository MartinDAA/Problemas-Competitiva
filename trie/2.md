# Problema 2 Trie: Longest Word in Dictionary

## Link
[Leetcode - 720. Longest Word in Dictionary](https://leetcode.com/problems/longest-word-in-dictionary/description/)

## Enunciado
Given an array of strings `words` representing an English Dictionary, return *the longest word in `words` that can be built one character at a time by other words in `words`.*

If there is more than one possible answer, return the longest word with the smallest lexicographical order. If there is no answer, return the empty string.

Note that the word should be built from left to right with each additional character being added to the end of a previous word. 

Example 1:

Input: `words = ["w","wo","wor","worl","world"]`\
Output: `"world"`\
Explanation: The word "world" can be built one character at a time by "w", "wo", "wor", and "worl".

Example 2:

Input: `words = ["a","banana","app","appl","ap","apply","apple"]`\
Output: `"apple"`\
Explanation: Both "apply" and "apple" can be built from other words in the dictionary. However, "apple" is lexicographically smaller than "apply".
 

Constraints:

`1 <= words.length <= 1000`\
`1 <= words[i].length <= 30`\
`words[i]` consists of lowercase English letters.

## Explicación

### Enfoque

Se busca la palabra más larga que se pueda formar añadiendo una letra a la vez, de manera que cada prefijo intermedio también sea una palabra del diccionario. La solución utiliza un **trie** para almacenar todas las palabras y una **búsqueda en profundidad (DFS)** que solo explora caminos válidos, es decir, aquellos donde cada nodo visitado corresponde a una palabra completa.

### Algoritmo

1. **Construcción del trie (función `add`)**  
   - Se inserta cada palabra en el trie carácter por carácter.  
   - En el nodo correspondiente al final de cada palabra se guarda su índice (`endOf`). Los nodos intermedios no tienen índice.

2. **Búsqueda DFS (función `dfs`)**  
   - Se realiza un recorrido en profundidad sobre el trie, comenzando desde la raíz.  
   - **Condición de exploración**: un nodo solo se procesa si es la raíz o si tiene un `endOf` válido (es decir, representa una palabra completa). Esto asegura que solo se consideren prefijos que sean palabras del diccionario.  
   - Para cada nodo válido, se obtiene la palabra correspondiente (usando `words[node->endOf]`) y se compara con la mejor respuesta actual según:
     - Mayor longitud.
     - En caso de empate, menor orden lexicográfico.
   - Luego, se exploran todos los hijos del nodo actual (letras `'a'` a `'z'`) que existan en el trie, apilándolos para continuar el DFS.

3. **Selección de la respuesta**  
   - La variable `ans` se actualiza con la palabra que cumple los criterios de desempate. Al final del recorrido, se devuelve `ans`.

### Complejidad

- **Tiempo**: `O(n * L)`, donde `n` es el número de palabras y `L` es la longitud máxima de una palabra, para construir el trie. El DFS visita cada nodo del trie una sola vez, por lo que la complejidad total es lineal en el número total de caracteres de todas las palabras.
- **Espacio**: `O(n * L)` para almacenar el trie.

### Puntos clave

- La restricción de que cada prefijo debe ser una palabra se modela naturalmente con el trie, ya que solo se avanza a través de nodos que son finales de palabra.
- El uso de DFS (implementado con una pila iterativa) permite recorrer todas las palabras válidas sin necesidad de recursión, evitando posibles desbordamientos de pila.
- El criterio de desempate lexicográfico se aplica directamente al comparar las cadenas con el operador `<` de C++.
