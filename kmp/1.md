# Problema 1 Algoritmo KMP: 1137B. Camp Schedule

## Link

[Codeforces - 1137B. Camp Schedule](https://codeforces.com/contest/1137/problem/B)

## Enunciado

B. Camp Schedule

Input file:	standard input\
Output file: standard output\
Time limit:	1 second\
Memory limit: 512 megabytes
    
The new camp by widely-known over the country Spring Programming Camp is going to start soon. Hence, all the team of friendly curators and teachers started composing the camp's schedule. After some continuous discussion, they came up with a schedule s, which can be represented as a binary string, in which the i-th symbol is '1' if students will write the contest in the i-th day and '0' if they will have a day off.

At the last moment Gleb said that the camp will be the most productive if it runs with the schedule t (which can be described in the same format as schedule s). Since the number of days in the current may be different from number of days in schedule t, Gleb required that the camp's schedule must be altered so that the number of occurrences of t in it as a substring is maximum possible. At the same time, the number of contest days and days off shouldn't change, only their order may change.

Could you rearrange the schedule in the best possible way?

Input

The first line contains string s (1⩽|s|⩽500000), denoting the current project of the camp's schedule.

The second line contains string t (1⩽|t|⩽500000), denoting the optimal schedule according to Gleb.

Strings s and t contain characters '0' and '1' only.

Output

In the only line print the schedule having the largest number of substrings equal to t. Printed schedule should consist of characters '0' and '1' only and the number of zeros should be equal to the number of zeros in s and the number of ones should be equal to the number of ones in s.

## Explicación

### Idea principal

Tenemos una cadena `s` cuyos caracteres podemos reordenar libremente, y un patrón `t`. Queremos construir una nueva cadena `ans` (usando exactamente los mismos `0`s y `1`s que hay en `s`) que contenga el **máximo número posible de ocurrencias** de `t` como subcadena contigua.

La clave está en que, para maximizar las ocurrencias, debemos colocar las copias de `t` lo más "pegadas" posible. Al colocar una copia de `t` justo después de otra, la **superposición máxima** que podemos permitir sin romper la periodicidad es el **borde propio más largo** de `t` (es decir, el prefijo que también es sufijo).

### Solución

1.  **Contar caracteres disponibles:** Contamos la cantidad de `0`s y `1`s en `s`.
2.  **Calcular la superposición:** Usamos la **función prefijo de KMP** sobre `t` para obtener `pi[m-1]`, que es la longitud del borde más largo de `t`. Llamemos a este valor `overlap`.
3.  **Construcción greedy:**
    - Primero, intentamos colocar una copia **completa** de `t`. Si en algún momento nos falta algún carácter (no hay suficientes `0` o `1`), significa que ni siquiera podemos formar una copia. En ese caso, agregamos todos los caracteres restantes a la respuesta y terminamos.
    - Si logramos colocar el primer `t` completo, a partir de ahora, en lugar de colocar `t` entero otra vez, solo necesitamos colocar el **sufijo** `t[overlap ... m-1]`. Esto se debe a que el prefijo de longitud `overlap` de la nueva copia ya está presente como sufijo de la copia anterior, gracias a la superposición.
    - Repetimos este paso (colocar el sufijo `t[overlap...]`) hasta que nos falte algún carácter para completar el siguiente sufijo. En ese momento, agregamos los caracteres restantes (da igual el orden) y finalizamos.

**Algoritmo utilizado:** KMP (función prefijo) únicamente para obtener el borde más largo de `t`.

**Complejidad:** `O(|s| + |t|)`.
