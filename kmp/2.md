# Problema 2 Algoritmo KMP: 432D. Prefixes and Suffixes

## Link

[Codeforces - 432D. Prefixes and Suffixes](https://codeforces.com/problemset/problem/432/D)

## Enunciado
D. Prefixes and Suffixes

Input file: stdin\
Output file: stdout\
Time limit: 1 second\
Memory limit: 256 megabytes

You have a string s = s1s2...s|s|, where |s| is the length of string s, and si its i-th character.

Let's introduce several definitions:

* A substring s[i..j] (1 ≤ i ≤ j ≤ |s|) of string s is string sisi + 1...sj.
* The prefix of string s of length l (1 ≤ l ≤ |s|) is string s[1..l].
* The suffix of string s of length l (1 ≤ l ≤ |s|) is string s[|s| - l + 1..|s|].

Your task is, for any prefix of string s which matches a suffix of string s, print the number of times it occurs in string s as a substring.

Input

The single line contains a sequence of characters s1s2...s|s|(1 ≤ |s| ≤ 105) — string s. The string only consists of uppercase English letters.

Output

In the first line, print integer k (0 ≤ k ≤ |s|) — the number of prefixes that match a suffix of string s. Next print k lines, in each line print two integers li ci. Numbers li ci mean that the prefix of the length li matches the suffix of length li and occurs in string s as a substring ci times. Print pairs li ci in the order of increasing li.

## Explicación

### Idea principal

Dada una cadena `s`, debemos encontrar todos los prefijos que también son sufijos (conocidos como "bordes"). Para cada uno de esos bordes, debemos imprimir su longitud y el **número total de veces que ese prefijo aparece como subcadena** en toda la cadena `s` (incluyendo las ocurrencias superpuestas).

La dificultad no está en encontrar los bordes, sino en contar sus ocurrencias de forma eficiente.

### Solución

1.  **Encontrar todos los bordes:** Calculamos el arreglo de prefijos `pi` de KMP para `s`. El borde más largo de toda la cadena es `pi[n-1]`. Para obtener todos los bordes (más cortos), simplemente recorremos la cadena de fallos:
    `len = pi[n-1]`
    `while (len > 0) { guardar len; len = pi[len-1]; }`
    A esto le añadimos la longitud `n` (la cadena completa siempre es prefijo y sufijo de sí misma).

2.  **Contar las ocurrencias de cada prefijo:** Aquí está el truco especial (una pequeña modificación sobre el KMP estándar):
    - Inicializamos un arreglo `cnt` de tamaño `n+1` con `1` en las posiciones `0` a `n-1`. Es decir, `cnt[i] = 1` para cada prefijo que termina en la posición `i` (cada prefijo aparece al menos una vez como sí mismo).
    - Procesamos las posiciones desde `i = n-1` hacia abajo hasta `0`. Para cada `i`, sumamos `cnt[i]` a `cnt[pi[i] - 1]`.
    - ¿Por qué funciona? `pi[i]` es la longitud del borde más largo del prefijo `s[0..i]`. Ese borde es a su vez un prefijo más corto que aparece terminando en la misma posición `i`. Al sumar las ocurrencias del prefijo más largo a su borde, estamos propagando la frecuencia de los prefijos grandes hacia los prefijos pequeños que los contienen como sufijo. Al terminar el barrido, `cnt[L-1]` contiene el número total de veces que el prefijo de longitud `L` aparece en toda la cadena.

3.  **Imprimir resultados:** Para cada borde `L` obtenido en el paso 1, imprimimos `L` y `cnt[L-1]`. El borde de longitud `n` (la cadena completa) siempre tiene conteo `1`.

**Algoritmo utilizado:** KMP (función prefijo) para obtener los bordes y una **DP de propagación** sobre el arreglo `pi` para contar las ocurrencias (esta técnica es muy común en problemas de KMP).

**Complejidad:** `O(n)`.
